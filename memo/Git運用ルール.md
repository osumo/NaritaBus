# かんたん Git/GitHub つかいかたガイド

チームみんなでコードを共有するための、最低限のルールです。
「壊すのが怖い」と思わずに、この順番通りに進めれば大丈夫です！

## 0. 準備：VS Code を開く
まずは、作業をしやすくするために VS Code を起動します。

1. **Eclipse で操作**
   - プロジェクト（NaritaBus）を **右クリック** して、コマンドプロンプトを開く
2. **VS Code を起動**
   - 黒い画面が出たら、`code .` と打って Enter を押します。
   - VS Code が開いたら準備完了です！

---

## 1. VS Code のボタンで操作する方法（おすすめ）
コマンドを打たなくても、左側のアイコンからマウス操作で Git が使えます。

1. **最新にする**: 
   - 左下のブランチ名（main）を確認し、青い更新マーク（または「Sync Changes」）を押して最新を取り込みます。
2. **枝を作る**: 
   - ブランチ名をクリックして「新しいブランチの作成」を選びます。
   - `feature/名前-やったこと` と入力します。
3. **保存（コミット）**: 
   - 左側の「ソース管理」アイコン（枝分かれしたアイコン）を押します。
   - 変更したファイルの横の **[+]** ボタンを押します（ステージング）。
   - 上の入力欄にメッセージ（例: ログイン作成）を書いて **[コミット]** ボタンを押します。
4. **提出（Push）**: 
   - **[変更の同期]** または **[発行]** ボタンを押して GitHub に送ります。

---

## 2. コマンドで操作する方法（なれてきたら）
VS Code 下側の「ターミナル」に打ち込んで操作することもできます。

### 準備：最新の状態にする
```bash
git checkout main
git pull origin main
```

### 出発：自分専用の「枝」を作る
```bash
git checkout -b feature/名前-やったこと
```

### 保存：変更を記録する
```bash
git add .
git commit -m "メッセージ内容"
```

### 提出：GitHubに送る
```bash
git push origin feature/名前-やったこと
```

---

## 3. 合流：みんなのコードに混ぜる（Webで操作）
GitHubの画面を開くと、「Compare & pull request」というボタンが出るので、それを押します。
1. **Pull Requestを作る**: 「これを作ったので確認してください！」と送る。
2. **OKをもらう**: 仲間が確認してOKが出たら、「Merge pull request」を押して合流！
