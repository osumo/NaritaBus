# かんたん Git/GitHub つかいかたガイド

チームみんなでコードを共有するための、最低限のルールです。
「壊すのが怖い」と思わずに、この順番通りに進めれば大丈夫です！

## 0. 準備：VS Code を開く
まずは、作業をしやすくするために VS Code を起動します。

1. **Eclipse で操作**
   - プロジェクト（NaritaBus）を **右クリック** して、コマンドプロンプトを開く
2. **VS Code を起動**
   - 黒い画面が出たら、`code .` と打って Enter を押します。
   - VS Code が開いたら準備完了です！

---

## 1. VS Code のボタン操作について
コマンドを打たなくても、左側のアイコンからマウス操作で Git が使えます。

### 最新の main ブランチの内容を取り込む方法
1. **ソース管理アイコン**を選択します。
2. **GitLens**の「>」をクリックして表示します。
   - ※表示されなかったらVS Codeを再起動してください（左上の「ファイル」→「最近使用した項目」からすぐに戻れます）。
3. **mainブランチ**であることを確認し、**altボタン**を押しながらクリックして、**「mainのプル」**を実行します。

### ブランチを作る方法
1. **GitLens**にマウスを置いて、**[BRANCHES]**を選択します。
2. **[Create branch]**を選択して、名前を入力します。
   - ブランチの名前は「名前-やったこと」（例：`masumura-ModifyMemo` など）にします。

### ブランチが完成した後の流れ
1. コミットを続けて、コードが完成したら、最新状態をメッセージにコミットします。
2. 他メンバーに確認のお願いと説明をします。
3. プルリクエストを承認してもらう、という流れになります。


---

## 2. コマンドで操作する方法（なれてきたら）
VS Code 下側の「ターミナル」に打ち込んで操作することもできます。

### 準備：最新の状態にする
```bash
git checkout main
git pull origin main
```

### 出発：自分専用の「枝」を作る
```bash
git checkout -b 名前-やったこと
```

### 保存：変更を記録する
```bash
git add .
git commit -m "メッセージ内容"
```


## 3. 合流：みんなのコードに混ぜる（Webで操作）
GitHubの画面を開くと、「Compare & pull request」というボタンが出るので、それを押します。
1. **Pull Requestを作る**: 「これを作ったので確認してください！」と送る。
2. **OKをもらう**: 他メンバが確認してOKが出たら、「Merge pull request」を押して合流！
