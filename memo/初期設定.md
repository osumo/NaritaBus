# 北海道物産ECサイト プロトタイプ初期設定ガイド

開発を始める前に、以下の手順で環境構築を行ってください。

## 0. プロジェクトの取得 (Gitクローン)
プロジェクトの最新コードを手元のパソコンにコピーします（初回のみ）。

1. GitHubのリポジトリページへ行き、**[Code]** ボタンを押してSSHを選択して、 URL をコピーします。
2. Eclipseで上部の　ファイルー＞インポートー＞Gitからプロジェクト（スマートインポート）ー＞クローンURIー＞URIにURLを貼り付け。
3. `NaritaBus` というフォルダが作成されたら完了です。Eclipse でこのフォルダをインポートしてください。

## 1. データベースの準備 (pgAdmin4)
1. pgAdmin4を開き、新しいデータベースを作成します。
   - データベース名: `NaritaBusDB`
   - （名前を変える場合は `application.properties` の `spring.datasource.url` も変更してください）

## 2. テーブル作成と初期データの投入
1. `src/main/resources/schema.sql` を開き、コメントアウト（`/* */`）を外せばテーブルが作成されます。
   - これにより、`users`, `products`, `favorites`, `orders` テーブルが作成されます。
2. 必要に応じて `src/main/resources/data.sql` のコメントアウトを外して実行し、テストデータを投入してください。
   - `application.properties` の設定により、次回起動時から自動実行されます。

## 3. アプリケーションのビルドと起動
1. プロジェクトを右クリックし、**「実行」→「Maven install」** を選択してビルドを行います。
2. `MybatisSample1Application.java` を右クリックし、**「実行」→「Spring Boot アプリケーション」** を選択して実行します。
3. サーバーを起動します。
4. 起動後、ブラウザで [http://localhost:8080/](http://localhost:8080/) にアクセスしてください。

## 4. Gitの初期設定
1. 初めてGitを使用する場合は、コマンドプロンプトやターミナルで以下の設定を行ってください。
   ```bash
   git config --global user.name "自分の名前"
   git config --global user.email "自分のメールアドレス"
   ```
2. 詳細は `memo/Git運用ルール.md` を参照してください。

## 5. 担当分担と役割
各コントローラーの冒頭にも記載されていますが、チーム内の役割分担は以下の通りです。

- **Aさん：ユーザー認証・登録基盤**
  - 会員登録、ログイン、ログアウト、セッション管理などの「土台」を担当。
- **Bさん：商品カタログ**
  - 商品一覧、フリーワード検索、フィルタリング、詳細表示の「カタログ機能」を担当。
- **Cさん：購買・配送フロー**
  - カート管理、注文入力、送料計算、注文完了メール送信などの「注文プロセス」を担当。
- **Dさん：ユーザー体験・エンゲージメント ☆**
  - プロジェクトの特色となる「付加価値機能」を横断的に担当。
  - マイページ、ランクシステム、育成要素、診断型検索、お気に入り機能、生産者へのチップ機能など。
